Getting Started

Welcome to the SecureKYC API documentation. Our REST API allows you to integrate seamless KYC verification into your platform.

Base URL

https://api.securekyc.com/v1

Quick Start

Follow these steps to get started with the SecureKYC API:

  1. Sign up for a developer account
  2. Get your API keys from the dashboard
  3. Make your first API call
  4. Handle the response

Your First API Call

cURL 
curl -X GET "https://api.securekyc.com/v1/status" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

Authentication

The SecureKYC API uses API keys for authentication. Include your API key in the Authorization header of every request.

API Key Authentication

Header 
Authorization: Bearer YOUR_API_KEY

Keep your API keys secure! Never expose them in client-side code or public repositories.

Rate Limiting

API requests are rate limited to ensure fair usage:

  • Starter: 100 requests per minute
  • Professional: 1000 requests per minute
  • Enterprise: Custom limits

Verification API

The verification API allows you to initiate and check the status of KYC verifications.

Initiate Verification

POST /verification/initiate

Request Body

JSON 
{
  "user_id": "user_12345",
  "callback_url": "https://yourapp.com/webhook",
  "verification_type": "full",
  "required_documents": ["government_id", "address_proof"]
}

Response

JSON 
{
  "verification_id": "ver_abcd1234",
  "status": "initiated",
  "verification_url": "https://verify.securekyc.com/ver_abcd1234",
  "expires_at": "2024-01-20T10:30:00Z"
}

Check Verification Status

GET /verification/{verification_id}

Response

JSON 
{
  "verification_id": "ver_abcd1234",
  "status": "completed",
  "result": "verified",
  "user_data": {
    "name": "John Doe",
    "email": "john@example.com",
    "verified_at": "2024-01-15T14:30:00Z"
  }
}

Consent Management

Manage user consent for data sharing across platforms.

Request Data Consent

POST /consent/request

Request Body

JSON 
{
  "user_id": "user_12345",
  "platform_id": "platform_xyz",
  "requested_data": ["personal_info", "address", "documents"],
  "purpose": "Account verification for banking services"
}

Retrieve User Data

GET /user/{user_id}/data

This endpoint requires valid consent from the user.

Query Parameters

Parameter Type Description
consent_token string Valid consent token from user
fields array Specific data fields to retrieve

Webhooks

Webhooks allow you to receive real-time notifications about verification status changes.

Webhook Events

  • verification.completed - Verification process completed
  • verification.failed - Verification failed
  • consent.granted - User granted data consent
  • consent.revoked - User revoked data consent

Webhook Payload

JSON 
{
  "event": "verification.completed",
  "data": {
    "verification_id": "ver_abcd1234",
    "user_id": "user_12345",
    "status": "verified",
    "timestamp": "2024-01-15T14:30:00Z"
  }
}

Error Handling

The API uses conventional HTTP response codes to indicate success or failure.

HTTP Status Codes

Code Description
200 OK - Request successful
400 Bad Request - Invalid request parameters
401 Unauthorized - Invalid API key
403 Forbidden - Insufficient permissions
404 Not Found - Resource not found
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Server error

Error Response Format

JSON 
{
  "error": {
    "code": "invalid_request",
    "message": "The request is missing required parameters",
    "details": {
      "missing_fields": ["user_id", "verification_type"]
    }
  }
}
